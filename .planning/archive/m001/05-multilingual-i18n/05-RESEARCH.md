# Phase 5: Multilingual (i18n) - Research

**Researched:** 2025-05-22
**Domain:** Internationalization (i18n) & Localization (L10n)
**Confidence:** HIGH

## Summary

This research identifies the optimal path for adding multilingual support to the Astrobiologia portal using Svelte 5 (runes) and SvelteKit 2. The primary recommendation is to use **Paraglide-js** as the i18n engine due to its native Svelte 5 support and superior SEO capabilities through localized URL routing.

For article content, we will adopt a **Relational Translation Pattern** in Appwrite, separating common metadata from localized content. DeepL will be integrated on the server-side to provide high-quality draft translations for the editorial team.

**Primary recommendation:** Use `paraglide-js` with localized URL routing (`/[lang]/`) and a separate `article_translations` collection in Appwrite.

| Capability                | Primary Tier          | Secondary Tier        | Rationale                                                                           |
| ------------------------- | --------------------- | --------------------- | ----------------------------------------------------------------------------------- |
| UI Translation            | Browser / Client      | Frontend Server (SSR) | Svelte 5 runes handle reactivity on client; SSR ensures initial load is translated. |
| Locale Detection          | Frontend Server (SSR) | Browser / Client      | Server detects locale from URL/Cookie/Headers for SEO and performance.              |

| Language Routing          | Frontend Server (SSR) | —                     | SvelteKit `reroute` hook manages localized paths for SEO.                           |

## Architecture Patterns

### Recommended Project Structure

```
src/
├── lib/
│   ├── paraglide/       # Auto-generated by paraglide
│   ├── appwrite.ts      # Updated to handle translations
│   └── components/
│       └── LanguageSwitcher.svelte
├── hooks.js             # Implement reroute hook
├── hooks.server.js      # Implement handle hook for locale detection
└── routes/
    └── articles/
        └── [slug]/      # Shared route for all languages
```

### Pattern 1: SvelteKit Localized Routing (SEO)

**What:** Use the `reroute` hook introduced in SvelteKit 2 to handle localized URLs without duplicating route files.
**When to use:** Always for public-facing content requiring SEO.
**Example:**

```typescript
// src/hooks.js
import { i18n } from "$lib/i18n";

export const reroute = i18n.reroute();
```

### Pattern 2: Relational Translation Schema (Appwrite)

**What:** Store translations in a separate collection linked to a master article.
**When to use:** When articles need to be translated into multiple languages but share metadata (image, category).

**Articles Collection:**

- `$id`: unique
- `featured_image`: file_id
- `category`: string
- `is_published`: boolean

**Article_Translations Collection:**

- `article_id`: relationship to Articles
- `language`: string (pt, en)
- `title`: string
- `content`: longtext
- `slug`: string (localized)

### Anti-Patterns to Avoid

- **Hardcoding Locales:** Never hardcode strings in components; always use paraglide functions.
- **Client-Side-Only i18n:** Avoid libraries that don't support SSR, as it breaks SEO for a news portal.
- **Flags for Languages:** Do not use flags (🇧🇷/🇺🇸) in UI; use native language names (Português / English).

## Don't Hand-Roll

| Problem               | Don't Build                 | Use Instead                   | Why                                                                  |
| --------------------- | --------------------------- | ----------------------------- | -------------------------------------------------------------------- |
| Localized URL Routing | Custom route logic          | `@inlang/paraglide-sveltekit` | Handles edge cases, trailing slashes, and `reroute` hooks perfectly. |
| HTML Translation      | Custom parser               | DeepL `tagHandling: 'html'`   | DeepL preserves HTML structure (Tiptap content) correctly.           |
| Date Formatting       | Manual `toLocaleDateString` | Paraglide/Intl.DateTimeFormat | Ensures month names and formats match the current locale perfectly.  |

## Common Pitfalls

### Pitfall 1: Flash of Untranslated Content (FOUC)

**What goes wrong:** The page loads in English, then flips to Portuguese after hydration.
**Why it happens:** Locale detection happens only on the client.
**How to avoid:** Use SvelteKit `hooks.server.js` to detect locale from the URL or cookies and pass it to the initial HTML via Paraglide.

### Pitfall 2: Broken SEO (hreflang)

**What goes wrong:** Google doesn't know the English version of a Portuguese article exists.
**Why it happens:** Missing `<link rel="alternate" hreflang="...">` tags.
**How to avoid:** Use Paraglide's `generateStaticLocalizedUrls` to inject these tags in the root `+layout.svelte`.

## Code Examples

### DeepL Integration (Server Action)

```typescript
// src/routes/admin/artigos/[id]/+page.server.ts
import * as deepl from "deepl-node";

const translator = new deepl.Translator(DEEPL_AUTH_KEY);

export const actions = {
  translate: async ({ request }) => {
    const data = await request.formData();
    const content = data.get("content") as string;
    const targetLang = data.get("targetLang") as deepl.TargetLanguageCode;

    const result = await translator.translateText(content, null, targetLang, {
      tagHandling: "html",
    });

    return { translatedText: result.text };
  },
};
```

### Language Switcher (Svelte 5)

```svelte
<script>
    import { i18n } from '$lib/i18n';
    import { page } from '$app/state';
</script>

<div class="flex gap-2">
    {#each i18n.locales as locale}
        <a
            href={i18n.resolveRoute(page.url.pathname, locale)}
            class="px-2 py-1 text-sm {i18n.locale === locale ? 'font-bold' : ''}"
        >
            {locale.toUpperCase()}
        </a>
    {/each}
</div>
```

## Open Questions

1. **Localized Slugs?** Should we use localized slugs (e.g., `/pt/buracos-negros` vs `/en/black-holes`)?
   - _Recommendation:_ Yes, for maximum SEO. The `article_translations` collection should store the localized slug.
2. **Free Tier DeepL?** The free tier is limited to 500k chars/month.
   - _Recommendation:_ Sufficient for initial launch, but must monitor usage in admin.
