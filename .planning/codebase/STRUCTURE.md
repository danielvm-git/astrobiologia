# Codebase Structure

**Analysis Date:** 2025-02-23

## Directory Layout

```
[project-root]/
├── components/          # Atom-level UI components (shadcn pattern)
├── messages/            # Translation source JSON files (Inlang)
├── project.inlang/      # Inlang/Paraglide configuration
├── scripts/             # Infrastructure and maintenance scripts
├── static/              # Public static assets
├── tests/               # Functional and integration tests
└── src/                 # Application source code
    ├── lib/             # Shared library code
    │   ├── components/  # Application-level Svelte components
    │   ├── paraglide/   # Generated localization runtime
    │   ├── server/      # Server-only logic and SDKs
    │   ├── stores/      # Svelte stores (legacy) or global state
    │   ├── appwrite.ts  # Shared Appwrite client and data helpers
    │   └── utils.ts     # General purpose utilities
    ├── routes/          # SvelteKit file-based routing
    ├── hooks.server.ts  # Server-side middleware
    └── hooks.ts         # Client-side middleware
```

## Directory Purposes

**`components/ui/`:**
- Purpose: Base UI components (atoms) like buttons, inputs, and cards.
- Contains: Svelte components with Tailwind 4 styling.
- Key files: `button.tsx`, `card.tsx`, `input.tsx`.

**`messages/`:**
- Purpose: Source of truth for static text translations.
- Contains: JSON files for each supported locale.
- Key files: `pt-br.json`, `en.json`.

**`src/lib/paraglide/`:**
- Purpose: Localization runtime generated by Paraglide-js.
- Contains: Locale detection logic and translation accessors.
- Key files: `runtime.js`, `server.js`, `messages.js`.

**`src/lib/server/`:**
- Purpose: Server-side logic that requires private environment variables.
- Contains: Administrative Appwrite client and session management.
- Key files: `appwrite.ts`.

**`src/routes/`:**
- Purpose: Application pages and API endpoints.
- Contains: SvelteKit `+page.svelte`, `+page.server.ts`, and `+layout.svelte`.
- Key files: `admin/`, `artigos/`, `categorias/`, `api/`.

## Key File Locations

**Entry Points:**
- `src/hooks.server.ts`: Handles localization and authentication before any route is rendered.
- `src/routes/+layout.svelte`: Root layout for the application.

**Configuration:**
- `svelte.config.js`: SvelteKit framework settings.
- `vite.config.ts`: Vite build and plugin (Paraglide, Tailwind) settings.
- `tsconfig.json`: TypeScript compiler options.

**Core Logic:**
- `src/lib/appwrite.ts`: Primary interface for data fetching from Appwrite.
- `src/lib/server/appwrite.ts`: Server-side Appwrite SDK initialization.

**Testing:**
- `tests/`: Integration tests organized by project phase (e.g., `phase1/`, `phase2/`, `phase5/`).

## Naming Conventions

**Files:**
- Svelte components: `PascalCase.svelte` (e.g., `ArticleCard.svelte`).
- Scripts: `kebab-case.js` (e.g., `setup-appwrite.js`).
- Tests: `name.test.ts`.

**Directories:**
- Feature folders: `kebab-case` (e.g., `admin-dashboard/`).
- Routes: `kebab-case` or SvelteKit parameters (e.g., `[slug]/`).

## Where to Add New Code

**New Feature:**
- Route: `src/routes/[feature]/`
- Components: `src/lib/components/` (high-level) or `components/ui/` (base).
- Tests: `tests/` in appropriate phase folder.

**New Translation:**
- Static text: Add to `messages/*.json`.
- Dynamic content: Add to `ArticleTranslation` schema in Appwrite.

**Utilities:**
- Shared helpers: `src/lib/utils.ts`.
- Server-side helpers: `src/lib/server/utils.ts`.

## Special Directories

**`src/paraglide/`:**
- Purpose: Generated message files for Paraglide.
- Generated: Yes
- Committed: No (usually ignored, though configuration varies).

**`.planning/`:**
- Purpose: GSD methodology documentation and codebase mapping.
- Generated: No
- Committed: Yes
