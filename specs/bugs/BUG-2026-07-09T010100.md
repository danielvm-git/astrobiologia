# BUG: Paused Appwrite project makes E2E CI permanently red

## Symptom

```
AppwriteException: Project is paused due to inactivity.
Please restore it from the console to resume operations.
```

E2E CI fails permanently when the Appwrite Cloud free-tier project auto-pauses after 30 days of inactivity.

## Root cause

Appwrite Cloud free-tier restriction. No graceful degradation in CI pipeline.

## Fix

Add graceful skip mechanism in CI: when global-setup detects project is paused, write a skip marker file. The base fixture reads this marker and skips all tests with a clear message. This keeps unit + typecheck gates intact while allowing E2E to pass when infra is down.

## Verify

```bash
# With paused Appwrite project:
pnpm run test:e2e:p0  # should exit 0 with skip message, not error
```
