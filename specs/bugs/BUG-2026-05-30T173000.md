# BUG-2026-05-30T173000: E2E tests pollute production site; hero shows imageless test article

## Problem

Two related issues reported on production (`https://astrobiologia.appwrite.network`):

### 1. E2E test runs add articles to the live site

- **Actual:** Each E2E run creates multiple **published** articles in the shared Appwrite database. They appear on the public homepage and `/artigos`. Fixture teardown deletes via HTTP but often fails silently; no global cleanup exists.
- **Expected:** Test artifacts should never appear on the production site after a test run.
- **Reproduce:**
  1. Run `pnpm test:e2e` with production Appwrite credentials in `.env`.
  2. Open the production site — new "Novo Artigo de Teste e2e-…" entries appear.

### 2. Featured images appear missing on homepage

- **Actual:** Homepage hero shows a broken-image placeholder (🔭) despite Albergaria articles having valid images in Storage.
- **Expected:** Hero and article cards show featured images from Appwrite Storage.
- **Reproduce:**
  1. Run E2E tests (seed article is `featured: true` with `featuredImage: ""`).
  2. Open production homepage — hero is the seed/test article with no image, pushing real articles aside.

## Root Cause Analysis

### Prior bug (recurrence context)

[BUG-2026-05-30T170200](./BUG-2026-05-30T170200.md) fixed `/preview` → `/view` URLs. That fix **is deployed and working** — production HTML uses `/view` URLs and Appwrite returns HTTP 200 for image files.

This report covers a **separate, data-layer issue**: test pollution and hero slot misconfiguration.

### E2E pollution — verified

**Phase 1 (Reproduce):** Queried production Appwrite — 22 published articles, 7 matching E2E patterns (`e2e-`, "Artigo de Teste", etc.). All test articles had `featuredImage: ""`.

**Phase 2 (Isolate):**

- `global-setup.ts` calls `ensureSeedPublishedArticle()` against the same Appwrite project as production (`.env` credentials).
- `createArticleViaUi` defaults to `status: "published"`.
- `base.fixture.ts` cleanup uses `page.request.delete` only — fails when session/auth unavailable after test.
- No `globalTeardown` existed.

**Phase 3 (Hypothesis):** E2E tests write published documents to the production database and cleanup is unreliable.

**Phase 4 (Verify):** Ran cleanup script — deleted 6 orphaned E2E articles. Confirmed test titles no longer appear on homepage. Risk: **Medium** (requires test infra changes).

### Hero imageless — verified

**Phase 1 (Reproduce):** Homepage hero was "Artigo de Teste (Semeado)" with 🔭 placeholder.

**Phase 2 (Isolate):** Seed article had `featured: true` and `featuredImage: ""`. `getFeaturedArticles()` picks featured articles ordered by `publishedAt`; seed/test articles occupied hero slot.

**Phase 3 (Hypothesis:** Seed article should not be featured; real Albergaria articles (with images) should rank in hero.

**Phase 4 (Verify):** After setting seed to `featured: false` and deleting test articles, production homepage HTML includes `/view` image URLs for Albergaria articles. Risk: **Low**.

### Images on article pages — verified working

- Albergaria articles have valid `featuredImage` file IDs in Appwrite.
- Production renders `/view` URLs (not `/preview`).
- Direct `curl -I` to image URLs returns HTTP 200.

## Resolution

### Changes implemented

1. **`appwriteTestClient.ts`**: Seed `featured: false`; `cleanupE2eArtifacts()` + `deleteE2eArticleById()`; patch existing seed on setup.
2. **`global-teardown.ts`**: Calls cleanup after every E2E run.
3. **`base.fixture.ts`**: Appwrite API delete in fixture teardown (HTTP fallback retained).
4. **`createArticleViaUi.ts`**: Default status `draft` (was `published`).
5. **`adminApiSession.ts`**: Tag test articles with `authorId: e2e-runner`, `featured: false`.
6. **`scripts/cleanup-e2e-articles.mjs`**: One-shot production cleanup.
7. **Production cleanup run**: Deleted 6 orphaned E2E articles; seed unpublished.

### Verification

- Production homepage HTML now contains `/view` image URLs for Albergaria articles.
- No `e2e-` test titles on homepage after cleanup.
- Unit tests: 64/64 passing.
