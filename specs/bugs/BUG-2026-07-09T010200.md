# BUG: Coverage thresholds provide no quality signal

## Symptom

Coverage gate always passes at 8% statements threshold. Actual coverage is 22.99%. The threshold is so low it cannot detect regressions.

## Root cause

Thresholds were set during early development as a baseline and never raised as coverage improved.

## Fix

Raise thresholds to meaningful levels that still pass:

- statements: 8 → 15
- branches: 45 → 50
- functions: 40 → 45

## Verify

```bash
pnpm --filter @astrobiologia/web-astro run test:coverage
# Should pass with new thresholds
```
