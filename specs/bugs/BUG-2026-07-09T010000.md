# BUG: \_\_dirname in ESM context causes CI failures

## Symptom

```
ReferenceError: __dirname is not defined in ES module scope
```

Hits `base.fixture.ts`, `global-setup.ts`, and `global-teardown.ts` randomly depending on how Playwright's `requireOrImport` resolves module format.

## Root cause

Playwright issue #37890: Playwright's ESM loader misidentifies files as CJS when `__dirname` global is present, but then loads them as ESM. Files using `path.resolve(__dirname, ...)` break intermittently.

## Fix

Replace `__dirname` with `import.meta.url` + `fileURLToPath()` + `path.dirname()` in:

- `tests/global-setup.ts`
- `tests/global-teardown.ts`
- `tests/fixtures/base.fixture.ts`
- `tests/helpers/appwriteTestClient.ts` (if applicable)

## Verify

```bash
npx playwright test --project=admin-crud --grep @p0
```

No `__dirname is not defined` errors.
