# BUG: Global vi.restoreAllMocks unmocks module-level mocks

## Symptom

`vitest.setup.ts` calls `vi.restoreAllMocks()` after each test. The `appwrite.test.ts` uses top-level `vi.mock("node-appwrite")`. If another test file needs the real module after appwrite.test.ts runs, behavior is unpredictable.

## Root cause

Module-level mocks are set once. `restoreAllMocks` may revert them. Currently not causing failures only because no other test file depends on the real `node-appwrite` after appwrite.test.ts runs.

## Fix

Replace top-level `vi.mock` with per-suite mock setup using `beforeAll`/`afterAll`, or add `mockRestore: false` to `vi.mock` call.

pnpm --filter @astrobiologia/web-astro run test:unit
