# BUG-2026-05-30T190400: metaDescription validation error on article update

## Problem

When updating an article translation via `PUT /api/admin/articles/[id]`, Appwrite rejects the request with:

```
AppwriteException: Invalid document structure: Attribute "metaDescription" has invalid type.
Value must be a valid string and no longer than 500 chars
```

**Actual behavior:** Article edits fail. The error occurs at line 147 of `src/pages/api/admin/articles/[id].ts` when calling `databases.updateDocument()`.

**Expected behavior:** Article translation updates succeed and persist without validation errors.

**Reproduce:**

1. Open the admin article editor
2. Edit any article translation without filling in the `metaDescription` field
3. Save the article
4. Observe the error in the server logs and UI

## Root Cause Analysis

**Phase 1 — Reproduce:** Confirmed via error logs that updating articles with empty/missing `metaDescription` fails consistently.

**Phase 2 — Isolate:**
The code path for translation updates is:

1. Line 108-122: Loop through `translationsInput[]`
2. Line 109: `const clean = sanitize(t)` — filters out `undefined` values
3. Line 142-149: Call `databases.updateDocument()` with the cleaned object

The `sanitize()` function (lines 17-24) explicitly removes `undefined` values:

```typescript
return Object.fromEntries(
  Object.entries(data).filter(
    ([k, v]) => !k.startsWith("$") && !forbidden.has(k) && v !== undefined
  )
);
```

So when `metaDescription` is `undefined` in the input:

- It's filtered out by `sanitize()` → not in `clean` object
- At line 147: `clean["metaDescription"]` evaluates to `undefined`
- Appwrite receives `{ ..., metaDescription: undefined }`
- Appwrite treats `undefined` as an invalid type (not a valid string)

**Phase 3 — Hypothesis:** When `metaDescription` is missing from the request body or is explicitly `undefined`, the code sends `undefined` to Appwrite instead of a valid string (empty string or existing value). Appwrite validates that string attributes must be actual strings, not `undefined` or `null`.

**Phase 4 — Verify:**

- Test fixtures set `metaDescription: undefined` (line 47 of `tests/fixtures/articles.ts`)
- Type definition requires `metaDescription: string` (line 10 of `src/lib/article-editor-types.ts`)
- API code sends `undefined` directly: `metaDescription: clean["metaDescription"]` (line 147)
- Appwrite schema enforces `string` type with max 500 chars

**Risk level: Medium** — affects all article translation updates when metadata is empty; requires careful fix to avoid breaking existing valid updates.
